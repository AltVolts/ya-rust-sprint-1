# **Библиотечный крейт  `parser`**

## Парсер форматов банковских записей

Библиотека `parser` предоставляет набор инструментов для чтения (парсинга) и записи (сериализации) данных 
о банковских транзакциях в различных форматах. Основной принцип — использование стандартных трейтов ввода/вывода `Read` и `Write`, 
что позволяет работать с любыми источниками и приёмниками данных без привязки к конкретным типам (файлы, стандартный ввод, буферы в памяти и т.д.).

## Возможности

- Поддержка нескольких форматов: бинарный (спецификация YP Bank), CSV, JSON.
- Единый интерфейс для чтения/записи благодаря трейтам `std::io::Read` и `std::io::Write`.


## Использование

Добавьте зависимость в `Cargo.toml`:

```toml
[dependencies]
parser = { path = "../parser" }
```

## Примеры 
В директории ```examples/``` находятся файлы с примерами использования, 
которые можно запустить из директории проекта "Parser:
```bash
cargo run --example example_bin_parser
```
```bash
cargo run --example example_csv_parser
```
```bash
cargo run --example example_txt_parser
```
Результаты примеров буду записаны в директорию ```write_test_files/```
